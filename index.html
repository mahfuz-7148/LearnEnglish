Frequently Asked Questions

1. What is the difference between var, let, and const?
var => Function-scoped, hoisted, re-declarable, re-assignable. let => Block-scoped, not hoisted, re-assignable but not re-declarable. const => Block-scoped, not re-assignable.
2. What is the difference between map(), forEach(), and filter()?
map() => Creates new array with transformed elements. forEach() => Executes function for each element, no return. filter() => Creates new array with elements passing a test.
3. What are arrow functions, and how are they different from regular functions?
Arrow Functions => Concise syntax, no arguments Differences => Cannot be used as constructors, more concise syntax compared to regular functions.
4. How do JavaScript Promises work?
Promises handle async operations with states such as Pending, Fulfilled, Rejected. Use resolve or reject in new Promise(), handle with .then(), .catch().
5. How does closure work in JavaScript?
Closure is when a function retains access to its outer scope's variables after the outer function finishes and enabling data privacy
